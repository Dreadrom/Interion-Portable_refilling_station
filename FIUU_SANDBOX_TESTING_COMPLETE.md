Using Postman or curl:

```bash
# Create Payment Request
curl -X POST https://sandbox-pg.fiuu.com/payment/create \
  -H "Content-Type: application/json" \
  -d '{
    "merchant_id": "SB_bluediesel",
    "amount": "100.00",
    "order_id": "TEST-'$(date +%s)'",
    "bill_name": "Test User",
    "bill_email": "test@bluediesel.com.my",
    "bill_mobile": "60123456789",
    "bill_desc": "Wallet Top-Up",

    "channel": "FPX_B2C",
    "returnurl": "https://your-app.com/payment/return",
    "callbackurl": "https://your-app.com/payment/callback"
```

## ⚠️ Important Findings

### 1. Cloudflare Protection
- Portal has CAPTCHA after session timeout
- Cannot automate repeated logins
- Manual login required periodically

### 2. Session Management
- Portal sessions expire after ~30 minutes
- Need to re-login to access dashboard
- API keys remain valid

### 3. Payment Timeouts
- Default QR expiry: ~15 minutes (needs confirmation from Fiuu)
- Recommend requesting 30+ minute expiry
- Add countdown timer in app

### 4. Polling Recommendations
- Interval: 2-3 seconds
- Max attempts: 60 (2 minutes at 2s interval)
- Implement exponential backoff on errors

### 5. Webhook Signature
- Use Secret Key to validate webhook
- Prevent replay attacks
- Log all webhook attempts

## 🎓 Best Practices from Testing

### 1. Error Handling
```javascript
// Handle all payment statuses
switch (paymentStatus) {
  case '00': // SUCCESS
    updateWalletBalance(amount);
    showSuccessScreen();
    break;
  case '11': // FAILED
    showErrorScreen('Payment failed. Please try again.');
    break;
  case '22': // PENDING
    continuePolling();
    break;
  case 'CANCELLED':
    returnToTopUpScreen();
    break;
  case 'EXPIRED':
    showErrorScreen('QR code expired. Please generate a new one.');
    break;
  default:
    logUnknownStatus(paymentStatus);
    showErrorScreen('Unknown payment status. Contact support.');
}
```

### 2. User Experience
- Show loading indicators during polling
- Display countdown timer on QR code
- Provide manual refresh button
- Show transaction history
- Send confirmation notifications

### 3. Security
- Never store Secret Key in frontend
- Always validate webhook signatures
- Use HTTPS for all API calls
- Sanitize user inputs
- Log all transactions

### 4. Testing Strategy
- Test all payment methods in sandbox
- Test all error scenarios
- Test app closure during payment
- Test network interruptions
- Test concurrent payments
